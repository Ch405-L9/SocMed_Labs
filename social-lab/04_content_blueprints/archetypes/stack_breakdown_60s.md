# stack_breakdown_60s

## Series: Phase 1 Daily Ops

**Purpose:** Show the real daily work of running Phase 1 — builds, fixes, and documentation in Cursor, VS Code, or Chrome DevTools. Position Brandon as a working practitioner, not a talking head. Drive follows from developers and tech-adjacent business owners.

**ICP:** Solo attorneys, practice managers, and small business owners who are either technical enough to understand what they're seeing, or impressed enough by the confidence and speed to trust the work.

**Best Platforms:**
- YouTube Shorts (primary — education niche gets highest engagement rates, 9.5%)
- TikTok (primary — quick fix clips perform well with #webdev, #fullstack tags)
- LinkedIn (secondary — "how I run my day" ops content performs well with B2B owners)

**Ideal Length:** 30–50 seconds

**Visual Style:**
- Faceless screen-share of actual code editor or DevTools session
- Real-time typing preferred over sped-up playback (feels more authentic)
- Terminal or CLI shots add credibility
- Cursor IDE preferred (relevant to the tech-savvy audience and matches ad concept)
- No music required — silence + keyboard sounds or light electronic works

**Hook Patterns:**
- Start mid-task: "…and this is the CLS fix. One line of HTML."
- State the time constraint: "This is what I do in the 4:30 PM build block."
- Show the tool working: Open DevTools, run Lighthouse, read the score — then fix.

**Required Assets:**
- Cursor IDE (primary editor for screen-share)
- Chrome DevTools
- Terminal (bash or zsh)
- BADGR fix log (CSV or Notion — show logging the fix in real time)
- Optional: BADGR Phase 1 schedule screenshot for context overlay

**CTA Variants:**
- "Follow for a new fix every week."
- "Comment 'LOG' — I'll share the fix log template."
- "Save this for your next dev session."
- "This is Phase 1 daily ops — link in bio for the full workflow."
