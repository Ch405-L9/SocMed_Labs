8. 🧠 RAG INSTRUCTIONS (FOR CLAUDE CODE / YOUR PIPELINE)

## BADGRTECH RAG BEHAVIOR RULES

1. Always classify user intent first.
2. Match intent to registry router.
3. Load only the matched module file.
4. Apply BADGRTECH tone rules after retrieval.
5. If intent is ambiguous → default to core tone module.
6. If multiple matches → merge modules but prioritize enforcement rules first.
7. Never override enforcement policy unless explicitly requested.

## Retrieval Order
1. intent_router.json
2. module JSON file
3. enforcement rules (always applied last check)
4. output generation

## Tone Application Rule
Tone is not optional. It is a system constraint, not a stylistic choice.
