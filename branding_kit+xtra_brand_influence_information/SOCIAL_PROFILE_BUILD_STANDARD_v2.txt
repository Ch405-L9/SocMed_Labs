# 2. IMAGE HANDLING RULES (NON-DESTRUCTIVE — CRITICAL)

For ALL image assets, ONLY the following transforms are permitted:

  - Resizing to platform specifications
  - Format conversion (PNG → JPG if required by platform)
  - Light enhancement: contrast/clarity adjustment only — NO redesign
  - Cropping ONLY if required for platform ratio compliance

STRICT PROHIBITIONS:
  - No logo redesign
  - No color palette changes
  - No layout restructuring
  - No overlay additions

Every image asset block MUST include:

  "source_file"          — absolute path to original
  "transformations"      — array of transforms applied (resize, convert, crop, enhance)
  "processing_tool"      — tool used to execute transforms (default: imagemagick)
  "transform_command"    — exact executable command string
  "output_path"          — absolute path for final asset
  "compliance_confirmed" — true/false

IMAGE HANDLING RULES — MASTER VS PLATFORM OUTPUTS:

  MASTER FILE (official_badgr-logo_px512.png):
    - Transparent PNG. NEVER modify. NEVER add background. NEVER save with fill.
    - All platform outputs derive FROM this file. It stays untouched.

  PLATFORM OUTPUTS (profile photos, banners, covers):
    - Profile photos: resize logo, add brand-color padding bg (#0E305F), output JPG or PNG per platform spec.
    - Padding bg is REQUIRED for circle-crop platforms — prevents logo edge crop.
    - Banners/covers: navy canvas (#0E305F) + centered logo. Background is structural, not decorative.
    - These are DERIVED assets. The master is still transparent.

ImageMagick command pattern (profile photo with padding):

  convert <source> -resize <logo_WxH> -background '#0E305F' -gravity center -extent <canvas_WxH> <output>
