## What Dev and AI Need in Practice
To maximize accuracy and strictness of the build, ensure the following are ready and documented:

1. **Repository and Branching**
   - Confirm the main repo (`ReaderRSVP`) is cloned into a stable local path such as `/home/t0n34781/Projects/badgr_bolt`.
   - Establish a branch naming convention (for example `feature/billing`, `feature/cloud-sync`) and require feature branches plus pull requests for every change.[^20][^8]

2. **Environment and Secrets**
   - Environment variables: `ANTHROPIC_API_KEY`, and any Firebase or Play Console credentials, exported only in local config or CI secrets, never committed.[^6][^7]
   - `local.properties` and `.env` files excluded via `.gitignore`.

3. **AI Integration Settings**
   - For IDE or local tools that call models via Ollama, configure `qwen2.5-coder:7b` as the default model for code edits and `llama3.2:latest` or a Qwen omni model for design/analysis prompts.[^4][^5]
   - Provide a short project summary and links to key docs in the AI tool's "context" or "instructions" section so every session starts with the same constraints (privacy-first, open-core, Free vs Pro, etc.).[^21][^4]

4. **Specs and Tickets**
   - Break the remaining Phase 2 work into tickets with clear "done" criteria (for example, "billing flow completes purchase, entitlement persisted, test purchase on sandbox account").[^22][^23]
   - Keep a simple backlog file (`docs/tickets.md` or GitHub Issues) so both dev and AI always work from the same list of tasks.[^24][^25]

5. **Quality Gates**
   - Configure unit tests, lint, and static analysis as required steps before merging; treat all AI-suggested changes as untrusted until they pass tests and review.[^9][^6][^20]
   - For privacy and security-sensitive features (auth, sync, billing), create explicit threat/abuse considerations in the ticket descriptions so the assistant is prompted to handle edge cases.[^7][^6]

Putting these pieces in place gives your local dev and whichever model they call through Ollama or APIs a shared, consistent context and strict guardrails for completing BADGR Bolt.
